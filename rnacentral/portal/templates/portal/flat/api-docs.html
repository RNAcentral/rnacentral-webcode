{% markdown %} ## RNAcentral API Documentation Most data in RNAcentral can be accessed programmatically using RESTful API allowing for integration with other websites ## Browsable API The data is hyperlinked so that one can navigate the API without leaving the browser. This brings the API very close to being self-documented. Depending on whether the URL is viewed in the browser or is retrieved by a command line client, the response is either rendered as HTML or Json. RNAcentral provides versioned API browsable. ## Versioning To ensure that changes in the RNAcentral APIs don't break the third-party applications relying on it, the API is versioned, and the version is included in the API URL. For example, the current version is Version 1 and is available at /api/v1. No backward incompatible changes are made to each version after it's been made public. * deleting fields * changing urls An advance notice will be given before obsoleting an API version. The current API can also be accessed at /api/current, but it's not recommended to use this endpoint for long-term development as the underlying data model may change. ## Pagination options The response is paginated to prevent accidental downloads of large amounts of data and to speed up the API. The default pagination size is 10 entries, but it can be adjusted by specifying the page_size parameter. The maximum value of page_size is 100. ## Example * http://127.0.0.1:8000/api/v1/rna/?page_size=5 * http://127.0.0.1:8000/api/v1/rna/?page=2&page_size=5
